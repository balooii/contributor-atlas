# contributor-aliases.txt — canonical name + every known ID for each contributor.
# Read by merge.py to unify identities across git, GitLab, and Bugzilla before
# writing contributions.csv.
#
# One line per contributor (names are not unique — two lines with the same name
# are different people):
#   Name <id> [<id> ...]
#
# All identifiers on a line belong to the same person; order doesn't matter.
# ID types:
#   <real[at]email.com>   git commit email
#   <@gitlab-handle>   GitLab username
#   <#bugzilla-name>   Bugzilla display name
#
# An ID may carry a "|name" annotation — <id|Observed Name> — recording the
# contributor name the source data actually used for that ID. It is added
# (by make_alias_draft.py, on draft suggestions) only when that observed name
# differs from the canonical name at the start of the line, as a hint for why
# the ID was grouped here; an ID seen only under the canonical name gets no
# annotation. When one ID appeared under several different names, they are
# listed separated by "/": <id|Name One/Name Two>. Annotations are purely
# informational — merge.py ignores everything from "|" on — so they are safe
# to keep or delete.
#
# Workflow: make_alias_draft.py appends commented draft suggestions below this
# block, grouped by confidence. To accept one, just uncomment its line — the
# next run sorts it into the alphabetical block above, so don't worry about
# where it lands. If a draft lumps together IDs that actually belong to
# different people, don't uncomment it as-is; instead assign each ID to the
# right person by hand: add the IDs to that person's existing line above, or,
# if they aren't listed yet, add a new line with their Name.
#
# Unsure who an ID belongs to? Grep it (the email, @gitlab-handle, or
# #bugzilla-name) in contributions.csv to list that identity's contributions.
# Each row's leading contribution_id encodes the source and a reference you can
# chase down further: commit-<project>-<git-hash>, bugzilla-<project>-<bug-id>,
# or issue-<project>-<issue-or-MR-number>[-note-<id>]
#
